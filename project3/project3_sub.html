publishing

  • AOS 플러그인 적용하여 스크롤 시 움직임 구현
  • onmouseover="img1(this)" onmouseout="img2(this)", .setAttribute('src', '') 를 이용하여 mouseover시 이미지 변환
  • e.clientX, e.clientY 를 사용하여 마우스 커서를 따라다니는 요소 적용
  • requestAnimationFrame(); 을 적용하여 커서 대신하는 요소의 움직임 더욱 부드럽게 적용
  • :hover 시 transform: translateY 를 사용하여 호버시 이미지 움직임 적용
  • addClass 를 적용하여 a태그 호버시 마우스 커서 크기 변경
  • 메인 이미지 섹션에 swiper 적용하여 유니버셜 뮤직의 아티스트 이미지를 보여줌
  • 햄버거 메뉴바 toggleClass 적용하여 누를때 마다 스타일 변경
  • 햄버거 메뉴바에 j Query UI 를 적용하여 슬라이드 다운 스타일 구현

solution

리뉴얼 전 사이트의 문제점 파악

  • 메인 첫 화면의 이미지 픽셀 깨짐
  • 움직이는 요소의 부재로 사용자의 흥미를 끌 수 없다고 판단

리뉴얼 전 사이트의 문제점 해결

  • 용량, 화질이 적절한 이미지 사용
  • 웹사이트의 전체적인곳에 포인트컬러를 적용하여 통일성을 줌
  • 메인 첫 화면에 노래가 재생되는 듯한 막대 움직임을 표현하여 사용자의 흥미 유발

리뉴얼을 진행하며 생긴 문제점

마우스 커서를 따라다니는 요소 작업 시 따라다니는 속도가 느려 사용자가 사용하기에 불편함을 느낄 수 있음

노트북 1440px 에서 계속해서 움직이는 텍스트가 아랫부분이 잘려보임

리뉴얼을 진행하며 생긴 문제점 해결

requestAnimationFrame(); 을 사용하여 움직임이 더욱 부드럽고 반응성을 빠르게 해결함

media query 를 적용하여 노트북 환경에서도 글자가 잘려보이지 않도록 반응형웹 사이트로 해결함
