publishing

  • media="(min-width:768px),(min-width:1200px)" 값을 적용하여 모바일, 테블릿, pc버전 모두 적응 가능
  • main 이미지 h1에 animation 값을 적용하여 보다 동적인 메인 화면 구성
  • header h1::after 를 이용한 실선 디자인 요소 제작
  • header nav 에 justify-content: space-between을 사용하여 로고, 메뉴, 로그인의 간격 조정
  • display: flex가 사용되는 요소에 class명 flex를 사용하여 필요한 요소에 flex가 적용되도록 작성
  • 모바일 버전에서의 필요없는 요소는 display: none 을 적용하여 보다 깔끔하고 간결한 화면 제작
  • visibility: visible, visibillity: none 을 사용하여 display: none 의 특성인 요소가 사라지는 것이 아닌 숨김만을 필요로 하는 곳에 적용
  • news 섹션에서 모바일 버전은 text-overflow: ellipsis 사용하여 사용자에게 문장이 더 길다는 것을 인지 시켜줌
  • display: grid 와 gap: 을 적용하여 깔끔하고 스타일리쉬한 이미지 배치
  • grid-template-columns: repeat(3, 1fr) 을 적용하여 간결한 코드 작성 방식
  • main .history figure h6 요소에 white-space: nowrap 을 적용하여 텍스트의 줄바꿈을 방지
  • 카카오맵 지도는 div 생성후, sdk.js?appkey=(발급받은 APP KEY) 를 입력하여 카카오맵 지도 삽입

solution

리뉴얼 전 사이트의 문제점 파악

  • 사이트의 디자인이 다소 딱딱하고, 한림공원의 자연친화적인 느낌이 부족함
  • 포인트로 정해진 컬러가 없으며, 사용자의 흥미를 유발 할 수 있는 요소가 부족함

리뉴얼 전 사이트의 문제점 해결

  • 부드러운 animation 효과, hover, transition 값을 적용하여 해결
  • 웹사이트의 전체적인곳에 포인트컬러를 적용하여 통일성을 줌
  • 이미지에 display: grid를 적용하고 움직임을 표현하여 사용자의 흥미 유발

리뉴얼을 진행하며 생긴 문제점

history 섹션에 ::after 를 이용한 실선과 점을 제작하는 과정에서 실선과 점 사이에 margin 값을 필요로 하였지만 실선과 점이 교차하며 정확한 값 을 입력하기에는 다소 어려움이 생김.

또한 반응형 웹사이트를 제작하면서 모바일에서는 숨김이 필요한 요소에 display: none 값을 입력하면 빈 공간이 아닌 여백이 생겨 계획했던 디자 인을 벗어난 문제점.

리뉴얼을 진행하며 생긴 문제점 해결

실선은 width를 100% 값을 주며, 점(span) 에 부모(span) 을 추가하여 부모(span)에 background 값을 white(#fff) 을 입력하여 마진값이 생 긴것 처럼 표현 하였다. 모바일, 테블릿 환경에서도 구현 지장 없음.

display: none 을 입력시 요소가 아예 사라져 디자인이 무너지는 현상을 visibillity: none 을 사용하여 요소의 자리는 그대로 유지하되, 요소는 숨 길 수 있도록 해결함.
